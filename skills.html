Skills

This document describes what the concept of a 'skill' is, as used in the development of Nagwa content.

Table of Contents

What is a skill?

In simple terms, a skill is either:

  • a piece of information that the student must know and be able to recall
  • or a technique - something that the student is able to do as part of answering a question

In other words, a student has learned a skill if they know a fact or know how to do something.

Here are some examples of skills:

  • know that the Battle of Hastings took place in 1066
  • know what the circuit symbol for a resistor is
  • know the names of the elements that are in group one of the periodic table
  • be able to add two single digit numbers
  • be able to use the formula I = Q/t to find the current in a circuit, where Q is the charge that moves past a point in the circuit in a time t
  • be able to label the chambers of the heart on a diagram of the human heart

Why do we use 'skills'?

Skills are a convenient, consistent way of describing what a student should have learned by doing a lesson. They are also useful for describing what a student needs to know or be able to do before starting a particular lesson.

They are also useful on a question level. By listing the skills that a particular question tests or teaches, it becomes easy to see what the purpose of a question within a lesson is, and what makes that question different to other, perhaps similar, questions in the lesson.

How to write skills

Skills are used in lesson plans, and because of this, a skill should be able to follow on from the phrase 'By the end of this lesson, the student should' - i.e., the skill should complete that sentence.

  • A skill should start with a verb or verb phrase, such as
    • know
    • understand
    • be able to describe
    • be able to draw
    • be able to interpret
    • be able to calculate
    • ...
  • Since a skill should complete the above sentence, it shouldn't start with a capital letter.

Specificity vs. Simplicity

One of the tricky things about writing skills is how specific they should be versus how easy they are to read.

The list of skills for a lesson will be seen by video developers and other content developers, so it's important for the skills to be specific so that those people have a very clear idea of what is and isn't meant to be in the lesson.

However, being more specific often involves using more words. The list of skills covered by a lesson will also be seen by teachers using the Nagwa platform. If the list of skills is overly-wordy, this may discourage teachers from reading through the list.

So skills need to be written in a way that is sufficiently clear for everyone who's going to read them, but which doesn't over-explain them. This can be a difficult balance to get right.

Consider the following example skill:

  • know the symbols for all of the leptons

(A lepton is a type of fundamental particle.) This skill could be made more specific by explicitly stating each type of lepton:

  • know the symbol for the electron
  • know the symbol for the muon
  • know the symbol for the tauon
  • know the symbol for the electron neutrino
  • know the symbol for the mu neutrino
  • know the symbol for the tau neutrino

But this is an example of being too specific. Any video developer or teacher will know that these are the six types of leptons, and so it is not necessary to be this specific.

But this skill could also have been written in a way that's too vague. For example:

  • know all of the leptons

This is unclear. Know what about the leptons? A teacher reading this might think that the student also has to know the mass, charge, and spin of all of the leptons, which is not what was meant.
